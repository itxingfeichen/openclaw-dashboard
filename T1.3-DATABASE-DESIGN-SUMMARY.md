## 技术实现细节

### 数据库设计原则

1. **规范化设计**: 所有表满足第三范式（3NF）
2. **外键约束**: 确保引用完整性
3. **CHECK 约束**: 保证字段值有效性
4. **索引优化**: 为常用查询字段创建索引
5. **审计追踪**: 关键表包含 created_at/updated_at
6. **软删除支持**: 使用 status 字段而非物理删除

### 性能优化措施

#### SQLite 优化
```sql
PRAGMA journal_mode = WAL;           -- WAL 写入模式
PRAGMA synchronous = NORMAL;         -- 平衡安全和性能
PRAGMA cache_size = 10000;           -- 10000 页缓存
PRAGMA temp_store = MEMORY;          -- 临时存储用内存
PRAGMA mmap_size = 268435456;        -- 256MB 内存映射
```

#### 索引策略
- 主键：所有表使用 INTEGER PRIMARY KEY AUTOINCREMENT
- 唯一索引：username, email, task_id, token 等
- 外键索引：所有外键字段都有索引
- 查询索引：status, type, category 等常用过滤字段

### 安全特性

1. **密码加密**: bcrypt with salt rounds >= 10
2. **SQL 注入防护**: 参数化查询，禁止字符串拼接
3. **敏感数据标记**: configs.is_sensitive 字段
4. **会话管理**: token 唯一，expires_at 过期时间
5. **审计日志**: 记录所有关键操作
6. **角色权限**: admin/user/readonly 三级权限

### 可扩展性设计

1. **双数据库支持**: SQLite/PostgreSQL 抽象层
2. **迁移版本控制**: 支持增量升级和回滚
3. **Repository 模式**: 业务逻辑与数据访问分离
4. **配置驱动**: 所有配置通过 configs 表管理
5. **JSON 字段支持**: metadata, config_json 等灵活字段

## 使用示例

### 创建用户
```javascript
import { createUser } from './repositories/user-repository.js';
import bcrypt from 'bcryptjs';

const passwordHash = await bcrypt.hash('password123', 10);
const user = createUser({
  username: 'newuser',
  email: 'user@example.com',
  passwordHash,
  role: 'user'
});
```

### 创建任务
```javascript
import { createTask } from './repositories/task-repository.js';

const task = createTask({
  taskId: 'T1.3',
  title: '数据库设计',
  description: '设计并实现数据库结构',
  priority: 'critical',
  agent_id: 1,
  assigned_to: 1,
  due_date: '2026-03-14'
});
```

### 更新任务进度
```javascript
import { updateTaskProgress, addTaskLog } from './repositories/task-repository.js';

updateTaskProgress(task.id, 75);
addTaskLog(task.task_id, '完成数据库 Schema 设计', 'info');
```

### 查询 Agent 及其技能
```javascript
import { getAgentWithSkills } from './repositories/agent-repository.js';

const agent = getAgentWithSkills(agentId);
console.log(agent.skills); // ['feishu-doc', 'weather', ...]
```

## 注意事项

### ⚠️ 开发环境
- 默认使用 SQLite，数据存储在 `backend/data/openclaw.db`
- 测试使用独立数据库 `backend/data/test-openclaw.db`
- 定期清理旧数据避免文件过大

### ⚠️ 生产环境
- 必须使用 PostgreSQL
- 必须修改默认用户密码
- 配置合适的连接池大小
- 启用定期备份
- 监控数据库性能指标

### ⚠️ 数据安全
- 敏感配置使用 `is_sensitive = 1` 标记
- 密码必须使用 bcrypt 加密
- 所有用户输入必须参数化查询
- 定期审计敏感操作日志
