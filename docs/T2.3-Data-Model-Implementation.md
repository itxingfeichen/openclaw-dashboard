### 2.2 模型详情

#### User (用户)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String | @id @default(cuid()) | 主键，自动生成 |
| email | String | @unique | 用户邮箱，唯一索引 |
| password | String | - | 密码哈希 |
| name | String? | - | 用户名称，可选 |
| role | String | @default("user") | 用户角色 |
| createdAt | DateTime | @default(now()) | 创建时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

**关系**:
- 一对多 → Session (用户会话)
- 一对多 → Dashboard (用户仪表盘)

**索引**:
- `email` (唯一索引)

---

#### Session (会话)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String | @id @default(cuid()) | 主键，自动生成 |
| userId | String | - | 关联用户 ID (外键) |
| token | String | @unique | 会话令牌，唯一 |
| expiresAt | DateTime | - | 过期时间 |
| createdAt | DateTime | @default(now()) | 创建时间 |

**关系**:
- 多对一 → User (级联删除 onDelete: Cascade)

**索引**:
- `token` (唯一索引)

---

#### Dashboard (仪表盘)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String | @id @default(cuid()) | 主键，自动生成 |
| userId | String | - | 关联用户 ID (外键) |
| name | String | - | 仪表盘名称 |
| config | String? | @default("{}") | JSON 配置字符串 |
| createdAt | DateTime | @default(now()) | 创建时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

**关系**:
- 多对一 → User (级联删除 onDelete: Cascade)
- 一对多 → Widget (仪表盘组件)

**索引**:
- `userId` (普通索引)

---

#### Widget (组件)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String | @id @default(cuid()) | 主键，自动生成 |
| dashboardId | String | - | 关联仪表盘 ID (外键) |
| type | String | - | 组件类型 (chart/table/metric) |
| config | String? | @default("{}") | JSON 配置字符串 |
| position | String? | @default("{}") | 位置信息 (JSON) |
| createdAt | DateTime | @default(now()) | 创建时间 |

**关系**:
- 多对一 → Dashboard (级联删除 onDelete: Cascade)

**索引**:
- `dashboardId` (普通索引)

## 3. 技术实现

### 3.1 Prisma Schema 文件

**位置**: `/home/admin/openclaw-dashboard/backend/prisma/schema.prisma`

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String
  name      String?
  role      String   @default("user")
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  sessions  Session[]
  dashboards Dashboard[]

  @@map("users")
}

model Session {
  id        String   @id @default(cuid())
  userId    String
  token     String   @unique
  expiresAt DateTime
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@map("sessions")
}

model Dashboard {
  id        String   @id @default(cuid())
  userId    String
  name      String
  config    String?  @default("{}")
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user    User    @relation(fields: [userId], references: [id], onDelete: Cascade)
  widgets Widget[]

  @@map("dashboards")
}

model Widget {
  id         String   @id @default(cuid())
  dashboardId String
  type       String
  config     String?  @default("{}")
  position   String?  @default("{}")
  createdAt  DateTime @default(now())

  dashboard Dashboard @relation(fields: [dashboardId], references: [id], onDelete: Cascade)

  @@map("widgets")
}
```

```typescript
import { PrismaLibSql } from '@prisma/adapter-libsql';

const libsql = new PrismaLibSql({
  url: 'file:./prisma/dev.db',
});

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

  adapter: libsql,
});

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma;

export default prisma;
```

## 6. 使用示例

### 6.1 创建用户及关联数据

```typescript
import prisma from './src/db/prisma.js';

// 创建用户及其仪表盘和组件
const user = await prisma.user.create({
  data: {
    email: 'user@example.com',
    password: 'hashed_password',
    name: 'John Doe',
    role: 'admin',
    dashboards: {
      create: {
        name: 'System Overview',
        config: JSON.stringify({ theme: 'dark' }),
        widgets: {
          create: [
            {
              type: 'metric',
              config: JSON.stringify({ title: 'CPU Usage' }),
              position: JSON.stringify({ x: 0, y: 0, w: 6, h: 4 }),
            },
            {
              type: 'chart',
              config: JSON.stringify({ title: 'Memory' }),
              position: JSON.stringify({ x: 6, y: 0, w: 6, h: 4 }),
            },
          ],
        },
      },
    },
  },
  include: {
    dashboards: {
      include: {
        widgets: true,
      },
    },
  },
});
```

### 6.2 查询用户及其关联数据

```typescript
// 查询用户及其所有仪表盘和组件
const userWithDashboards = await prisma.user.findUnique({
  where: { email: 'user@example.com' },
  include: {
    sessions: true,
    dashboards: {
      include: {
        widgets: true,
      },
    },
  },
});

// 查询特定仪表盘及其组件
const dashboard = await prisma.dashboard.findUnique({
  where: { id: 'dashboard-id' },
  include: {
    widgets: {
      orderBy: { createdAt: 'asc' },
    },
  },
});
```

### 6.3 更新操作

```typescript
// 更新用户信息
const updatedUser = await prisma.user.update({
  where: { id: 'user-id' },
  data: {
    name: 'New Name',
    role: 'admin',
  },
});

// 更新仪表盘配置
const updatedDashboard = await prisma.dashboard.update({
  where: { id: 'dashboard-id' },
  data: {
    config: JSON.stringify({ theme: 'light', layout: 'flex' }),
    updatedAt: new Date(),
  },
});
```

### 6.4 删除操作 (级联)

```typescript
// 删除用户会自动删除其所有会话、仪表盘和组件
await prisma.user.delete({
  where: { id: 'user-id' },
});

// 删除仪表盘会自动删除其所有组件
await prisma.dashboard.delete({
  where: { id: 'dashboard-id' },
});
```

## 8. 安全考虑

### 8.1 密码存储

⚠️ **注意**: 当前 `User.password` 字段存储明文密码哈希。生产环境中必须:

1. 使用 bcrypt 或 argon2 进行密码哈希
2. 添加盐值 (salt)
3. 永远不要存储明文密码

示例:
```typescript
import bcrypt from 'bcryptjs';

const saltRounds = 12;
const hashedPassword = await bcrypt.hash(plainPassword, saltRounds);
```

### 8.2 会话令牌

- 使用加密安全的随机数生成器生成令牌
- 设置合理的过期时间 (建议 2 小时)
- 实现令牌刷新机制

### 8.3 级联删除

所有外键关系都配置了 `onDelete: Cascade`,确保:
- 删除用户时自动清理所有关联数据
- 删除仪表盘时自动删除所有组件
- 避免孤儿记录

## 9. 性能优化

### 9.1 索引策略

已配置的索引:
- `users.email` - 唯一索引 (登录查询)
- `sessions.token` - 唯一索引 (会话验证)
- `sessions.userId` - 普通索引 (关联查询)
- `dashboards.userId` - 普通索引 (用户仪表盘列表)
- `widgets.dashboardId` - 普通索引 (仪表盘组件列表)

### 9.2 查询优化建议

1. **使用 include 预加载关联数据**
   ```typescript
   // ✅ 推荐
   const user = await prisma.user.findUnique({
     where: { id },
     include: { dashboards: true },
   });
   
   // ❌ 避免 N+1 查询
   const user = await prisma.user.findUnique({ where: { id } });
   const dashboards = await prisma.dashboard.findMany({ where: { userId: id } });
   ```

2. **只选择需要的字段**
   ```typescript
   const user = await prisma.user.findUnique({
     where: { id },
     select: { id: true, email: true, name: true },
   });
   ```

3. **使用分页处理大数据集**
   ```typescript
   const dashboards = await prisma.dashboard.findMany({
     where: { userId },
     skip: (page - 1) * pageSize,
     take: pageSize,
   });
   ```
