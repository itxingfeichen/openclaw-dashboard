## 实现内容

### 1. JWT 认证模块 (`backend/src/auth/`)

#### jwt.js - JWT Token 生成和验证
- `generateAccessToken(payload)` - 生成访问令牌 (默认 1 小时过期)
- `generateRefreshToken(payload)` - 生成刷新令牌 (默认 7 天过期)
- `verifyToken(token)` - 验证令牌
- `decodeToken(token)` - 解码令牌 (不验证)
- `isTokenExpired(token)` - 检查令牌是否过期
- `getTokenExpiration(token)` - 获取令牌过期时间

#### password.js - 密码加密和验证
- `hashPassword(password)` - 使用 scrypt 算法加密密码
- `verifyPassword(password, hash)` - 验证密码
- `validatePasswordStrength(password)` - 密码强度验证
  - 最少 8 个字符
  - 包含大写字母
  - 包含小写字母
  - 包含数字
  - 包含特殊字符

#### middleware.js - 认证中间件
- `requireAuth` - 要求有效的 JWT 访问令牌
- `optionalAuth` - 可选认证 (有令牌则附加用户信息)
- `hasRole(...roles)` - 角色检查中间件
- `requireAdmin` - 仅管理员访问
- `rateLimiter` - 登录限流 (5 次/15 分钟/IP)

### 2. 用户服务层 (`backend/src/services/`)

#### auth-service.js - 认证服务
- `login(username, password, ipAddress, userAgent)` - 用户登录
- `logout(refreshToken)` - 用户登出
- `refreshAccessToken(refreshToken)` - 刷新访问令牌
- `getCurrentUser(userId)` - 获取当前用户信息
- `validateToken(token)` - 验证令牌

#### user-service.js - 用户服务
- `createUserAccount(userData)` - 创建用户
- `getUserByIdService(id, includeSensitive)` - 根据 ID 获取用户
- `getAllUsersService(options)` - 获取所有用户 (支持分页和搜索)
- `updateUserAccount(id, updateData)` - 更新用户
- `changePassword(id, currentPassword, newPassword)` - 修改密码
- `adminResetPassword(id, newPassword)` - 管理员重置密码
- `updateUserStatus(id, status)` - 更新用户状态
- `deleteUserAccount(id)` - 删除用户
- `userHasRole(userId, role)` - 检查用户角色
- `userIsAdmin(userId)` - 检查是否为管理员

### 3. API 路由 (`backend/src/routes/`)

#### auth.js - 认证路由
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/logout` - 用户登出
- `POST /api/auth/refresh` - 刷新 Token
- `GET /api/auth/me` - 获取当前用户
- `POST /api/auth/verify` - 验证 Token

#### users.js - 用户管理路由 (需要管理员权限)
- `GET /api/users` - 用户列表 (支持分页和搜索)
- `POST /api/users` - 创建用户
- `GET /api/users/:id` - 获取用户详情
- `PUT /api/users/:id` - 更新用户
- `PATCH /api/users/:id/status` - 更新用户状态
- `DELETE /api/users/:id` - 删除用户
