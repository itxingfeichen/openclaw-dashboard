### 2. 构建脚本

#### 根目录 package.json 脚本
- `npm run dev` - 同时启动前后端开发服务器
- `npm run build` - 构建前后端生产版本
- `npm run lint` - 运行代码检查
- `npm run format` - 格式化代码
- `npm run test` - 运行测试

#### Makefile 命令
- `make install` - 安装所有依赖
- `make dev` - 启动开发服务器
- `make build` - 构建生产版本
- `make lint` - 代码检查
- `make format` - 代码格式化
- `make test` - 运行测试
- `make clean` - 清理构建产物

### 3. 代码规范配置

#### ESLint
- **前端**: ESLint 9 + typescript-eslint + React 插件
- **后端**: ESLint 8 + @typescript-eslint/parser
- 配置文件: `frontend/.eslintrc.cjs`, `backend/.eslintrc.json`

#### Prettier
- 统一代码格式化规则
- 配置: `.prettierrc` (根目录), `frontend/.prettierrc`
- 忽略文件: `.prettierignore`

#### EditorConfig
- 统一编辑器配置（缩进、换行、字符集等）
- 配置: `.editorconfig`

## 📝 使用说明

### 快速开始

```bash
# 1. 进入项目目录
cd /home/admin/openclaw-dashboard

# 2. 安装所有依赖
make install
# 或
npm install && npm install --prefix frontend && npm install --prefix backend

# 3. 启动开发服务器
make dev
# 或
npm run dev

# 4. 访问应用
# 前端：http://localhost:8080
# 后端 API: http://localhost:3000
```

### 开发工作流

```bash
# 代码检查
make lint

# 代码格式化
make format

# 构建生产版本
make build

# 运行测试
make test
```

## ⚠️ 注意事项

1. **端口占用**: 前端默认使用 8080 端口，如被占用会自动切换
2. **TypeScript 类型**: 部分现有 .jsx 文件需要转换为 .tsx 以消除类型错误
3. **环境变量**: 后端需要配置 `.env` 文件（已有示例）
