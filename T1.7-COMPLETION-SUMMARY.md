## 技术实现细节

### Prometheus 指标

#### 默认指标（通过 prom-client 自动采集）
- `nodejs_version_info` - Node.js 版本
- `nodejs_eventloop_lag_seconds` - 事件循环延迟
- `nodejs_heap_size_used_bytes` - 堆内存使用
- `nodejs_heap_size_total_bytes` - 堆内存总量
- `process_cpu_seconds_total` - CPU 时间
- `process_resident_memory_bytes` - 常驻内存

#### 自定义 HTTP 指标
- `http_requests_total` - HTTP 请求总数
- `http_request_duration_seconds` - HTTP 请求延迟直方图
- `http_requests_in_flight` - 正在处理的请求数
- `api_response_time_milliseconds` - API 响应时间

#### 数据库指标
- `db_queries_total` - 数据库查询总数
- `db_query_duration_milliseconds` - 数据库查询延迟

#### 缓存指标
- `cache_hits_total` - 缓存命中数
- `cache_misses_total` - 缓存未命中数
- `cache_size_bytes` - 缓存大小

#### 错误指标
- `errors_total` - 错误总数（按类型和严重程度分类）


3. 验证端点：
```bash
curl http://localhost:8080/api/health
curl http://localhost:8080/api/health/metrics
```

### Prometheus 配置

添加以下配置到 `prometheus.yml`：

```yaml
scrape_configs:
  - job_name: 'openclaw-dashboard'
    static_configs:
      - targets: ['localhost:8080']
    metrics_path: '/api/health/metrics'
    scrape_interval: 10s
```

### Grafana 仪表板导入

1. 登录 Grafana
2. 导航到 Dashboards → Import
3. 上传 `monitoring/grafana-dashboard.json`
4. 选择 Prometheus 数据源
5. 点击 Import

### 告警规则导入

将 `monitoring/alert-rules.yml` 添加到 Prometheus 配置：

```yaml
rule_files:
  - "alert-rules.yml"
```
